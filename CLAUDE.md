## Architecture

### Data Flow

Raw Amazon review JSONL.GZ files → 4 pipeline stages → ML models:

1. **`build_user_counts`** — Downloads/streams reviews, computes per-user purchase counts per category → `data/processed/<Category>/user_counts_*.parquet`
2. **`filter_users`** — Aggregates into a global user-category matrix, scores users by `total_purchases × entropy_percentile`, keeps top 5% → `data/global/top_users.parquet`
3. **`extract_features`** — Streams reviews again keeping only top users, extracts per-review/user/item feature tables → `data/processed/<Category>/top_user_*.parquet`
4. **`create_sequences`** — Hash-shards users into 256 shards, builds temporal sequence samples (prefix → next category) → `data/global/sequence_samples_by_shard/`

### Registry-Driven I/O (`io/data_io.py`)

All data files are tracked in `configs/data_registry.yaml` (auto-generated by `io/registry_sync.py`). The main API:
- `ensure_local(namespace, key)` — resolves a registered file, auto-downloads from Google Drive via `gdown` if missing
- Registry is lazy-loaded once per process into module-level `_DATA_REGISTRY`
- Drive OAuth credentials cached at `configs/pydrive_credentials.json`

### Shard-Based Train/Val/Test Split

256 shards total (hash of `user_id`): first 204 = train (80%), next 26 = val (10%), last 26 = test (10%). Split is user-level, preventing temporal leakage. All models cap row counts (`MAX_TRAIN_ROWS=1M`, `MAX_VAL_ROWS=200K`, `MAX_TEST_ROWS=200K`).

### Model Pipeline Pattern

All models in `models/` follow the same pattern:
1. Load shards by split assignment
2. Apply `ColumnTransformer`: `SimpleImputer(median) → StandardScaler` for numeric; `SimpleImputer → OneHotEncoder` for categorical
3. Train sklearn estimator, evaluate with accuracy + top-3 accuracy + classification report

### Config (`utils/config.py`)

Single source of truth for all constants: data thresholds, shard counts, model hyperparams, URLs, row caps. All pipeline and model modules import from here.

### Path Resolution

`REPO_ROOT = Path(__file__).resolve().parents[3]` in every source module — all files live at depth `src/amazon_next_category/{subpackage}/{file}.py` (4 levels deep from root).
